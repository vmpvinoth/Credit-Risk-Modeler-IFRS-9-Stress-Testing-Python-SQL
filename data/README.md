# Data Directory

This folder holds all datasets used by the IFRS 9 Credit Risk notebook.

## Structure

```
data/
├── raw/           ← Place downloaded raw files here
└── processed/     ← Auto-generated by the notebook (do not edit)
```

---

## Required Files in `data/raw/`

### 1. LendingClub Loan Data (Primary Dataset)
**File:** `accepted_2007_to_2018Q4.csv` (extracted from `.csv.gz`)

| Property | Detail |
|----------|--------|
| Source | Kaggle — LendingClub |
| URL | https://www.kaggle.com/datasets/wordsforthewise/lending-club |
| Size | ~1.3 GB compressed / ~3.5 GB uncompressed |
| Records | ~2.26 million loans |
| Period | 2007 – 2018 Q4 |
| Key Columns | `loan_amnt`, `int_rate`, `grade`, `loan_status`, `annual_inc`, `dti`, `fico_range_low` |

**Download steps:**
```bash
# Via Kaggle CLI (requires kaggle account + API token)
kaggle datasets download -d wordsforthewise/lending-club
unzip lending-club.zip -d data/raw/
gunzip data/raw/accepted_2007_to_2018Q4.csv.gz
```

---

### 2. FRED Macroeconomic Data
**Files:** `fred_gdp.csv`, `fred_unrate.csv`, `fred_fedfunds.csv`

```bash
# Automated download (no account required)
python scripts/download_data.py
```

| File | FRED Series | Description |
|------|-------------|-------------|
| `fred_gdp.csv` | A191RL1Q225SBEA | Real GDP growth rate (quarterly %) |
| `fred_unrate.csv` | UNRATE | Unemployment rate (monthly %) |
| `fred_fedfunds.csv` | FEDFUNDS | Federal funds rate (monthly %) |

---

## Running Without Data

The notebook has a built-in **calibrated fallback** — all sections can be executed without downloading any data. The `DataLoader` class generates synthetic data calibrated to match real LendingClub statistics, so the modelling workflow, regulatory calculations, and outputs remain fully functional.

To use real data, simply place the files in `data/raw/` and re-run the notebook.

---

## Data Privacy

- The LendingClub dataset is **publicly available** and **anonymised** (no personal identifiers)
- FRED data is **public domain** (published by the US Federal Reserve)
- No proprietary or confidential data is used in this project
